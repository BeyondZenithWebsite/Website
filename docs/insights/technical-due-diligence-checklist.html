Due diligence checks that catch real risk

Updated with current benchmarks and practical implementation guidance for 10–100 person teams.

What is working now

  • Third-party and supply-chain exposure is now a major diligence lane.
  • Operational evidence (logs, runbooks, restore tests) is weighted higher than policy documents.
  • SMB targets are disproportionately exposed to common control gaps.

Evidence and benchmarks

  1. 2x increase in third-party linked breach share YoY (Verizon DBIR, 2025). Source
  2. ~4x SMBs targeted more than large organisations (DBIR, 2025). Source
  3. 66% Kubernetes production usage among surveyed end users (CNCF, 2023). Source
  4. 40% cite security as top container challenge (CNCF, 2023). Source

Execution playbook (next 30 days)

  • Test restore procedures and incident response with evidence, not claims.
  • Run mandatory third-party risk review for critical vendors and dependencies.
  • Prioritise patch latency, secrets handling, and access controls in first pass.

Data credibility note: Benchmarks are from reputable 2023–2026 sources where available; older baselines are included only when still industry-standard references.

Quick FAQ

Can this be implemented without a full platform rebuild?

Yes. Start with a scoped operating change and targeted technical fixes before major rewrites.

How soon should results show?

You should see directional improvement inside the first 2–4 weeks if cadence and ownership are enforced.
